# Product Mission

## Problem

Creators do not mainly need deeper worldbuilding software. They need a better way to **visualize and coordinate preproduction work** across multiple stages.

The current workflow is fragmented:

- ideas live in docs or chat
- scripts live in separate writing tools
- reference images live in folders or image generators
- storyboards live elsewhere, if they exist at all
- production handoff happens by manually restating decisions

This leads to three recurring pains:

### 1. Coordination Overload

AI can generate a lot, but creators still have to sort the output, decide what matters, and remember why something was chosen.

### 2. Poor Traceability

Important decisions disappear into chat logs, folders, and renamed files. Teams lose the chain from concept to scene to shot.

### 3. Weak Visualization Between Idea and Delivery

Most tools are either too unstructured, too admin-heavy, or too fragmented to support a coherent visual preproduction workflow.

## Core Insight

The real gap is not `AI generation`. The real gap is **human-centered coordination across preproduction stages**.

Vixio Studio should help creators answer:

- What stage are we in?
- What has already been decided?
- What is still missing?
- Which references and assets matter for this scene?
- How do we move from a brief or script into something visual and exportable?

## Product Thesis

Vixio Studio is an **AI-assisted project workspace** where creators can start from either a brief or a script, shape the work through stage-specific boards, keep canon and assets attached to that flow, and finish with deliverables that feel like the outcome of one coherent system.

## Primary Users

### Core Buyers

- Independent filmmakers
- Writer-directors
- Small production teams
- Animation creators
- Storyboard-heavy visual development teams

These users need more structure than generic docs or chat, but do not want enterprise production software.

### Adjacent Users

- Screenwriters developing toward visual treatment
- Narrative designers
- Worldbuilders who care more about visualization than encyclopedia depth
- Creative leads coordinating references across departments

## Value Proposition

| User Need | Vixio Studio Response |
|-----------|------------------------|
| I need to start from a concept | Brief-first planning flow |
| I already have a script | Script breakdown-first flow |
| I want AI help without losing control | AI coordinates and suggests; humans approve |
| I need visual clarity | Stage-aware boards and block lineage |
| I need supporting context | Canon and assets stay attached to the workflow |
| I need deliverables | Exports are built from the workflow, not bolted on |

## Why This Wins

### Better Than Worldbuilding Tools

Many creators do not want to maintain a world encyclopedia. They want to get to scenes, boards, and handoff-ready outputs faster.

### Better Than Production Admin Tools

Script breakdown and scheduling tools assume the creative work is already resolved. Vixio starts earlier and keeps the work visual.

### Better Than Chat + Image Tool Stacks

Chat plus image generation creates more output, but not more clarity. Vixio adds workflow state, traceability, and durable project memory.

## Product Boundaries

Vixio Studio is:

- a preproduction workspace
- a project memory system
- a visual coordination layer
- an AI-assisted handoff tool

Vixio Studio is not:

- a pure world wiki
- a generic database populated for its own sake
- a full production management suite
- an autonomous creator replacement

## Success Metrics

1. Creators can begin from either a brief or a script without being forced into worldbuilding.
2. Boards become the default place where preproduction work advances.
3. Canon and assets act as support layers rather than dominating the product identity.
4. Users export meaningful deliverables without copying decisions across tools.
5. Teams describe Vixio as helping them stay visually aligned and organized, not just helping them generate more AI output.
