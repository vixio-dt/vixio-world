## Market 1: Worldbuilding Platforms

### Major Players
World Anvil, Campfire, Obsidian, Legend Keeper, Kanka.io

### Pain Points to Exploit

| Pain Point | What Users Hate | Vixio Opportunity |
|------------|-----------------|-------------------|
| **Learning Curve** | BB codes, complex formatting | AI-native - just talk/paste |
| **Pricing Anxiety** | Subscriptions, feature paywalls | Simple model, data ownership |
| **Mobile/Offline** | Can't work anywhere | Progressive web app or local-first |
| **Privacy** | Free = public by default | Private by default |
| **Scalability** | Slow with large worlds | AI handles search/linking |
| **Manual Linking** | Hours maintaining connections | AI suggests/creates relationships |
| **Fragmentation** | 5 tools for one workflow | One tool that does the core well |

### Platform-Specific Issues

**World Anvil**
- Complex BB code formatting creates barrier to entry
- Free tier requires public content by default
- Performance degrades with large projects
- No lifetime option - ongoing subscription required
- Internet dependency restricts workflow

**Campfire**
- Modular pricing confusion - unclear feature costs
- Poor mobile/tablet optimization
- Collaboration requires all users have accounts
- Limited customization and public page control
- Scalability issues with large content volumes

**Obsidian**
- Technical setup complexity (plugins, GitHub, etc.)
- Sync costs extra ($8-10/month)
- Not purpose-built for worldbuilding

### What They All Lack

- **AI-native design** - AI is bolted on, not core
- **Flexible worldbuilding style** - They lean "hard" (structured)
- **Simplicity** - Feature bloat is common
- **True data portability** - Vendor lock-in
- **Visual asset focus** - Text-first, not asset-first

## Market 2: Live-Action Production Tools

### Production Export Gap

Traditional worldbuilding tools have limited production export:

| Platform | Screenplay | Storyboard | Shot List | Production Docs |
|----------|------------|------------|-----------|-----------------|
| World Anvil | ❌ | ❌ | ❌ | Basic HTML/PDF only |
| Campfire | ❌ | ❌ | ❌ | DOCX/PDF/RTF only |
| Plottr | ❌ | ❌ | ❌ | MS Word only |

**The Gap**: Worldbuilding tools focus on prose/narrative, not screen production workflows.

### 5 Specific Gaps to Target

| Gap | Problem | Vixio Solution |
|-----|---------|----------------|
| **No Screenplay Formatting** | Cannot auto-format dialogue, action lines, scene headings | AI converts narrative → properly formatted screenplay |
| **No Shot Planning** | Lack of camera direction, shot types, angle specs | AI suggests shots from scene content |
| **No Production Integration** | Missing connectivity to shot lists, call sheets, storyboards | Direct export to production formats |
| **Limited Visual Tools** | Basic text/picture support, no storyboard creation | **AI-generated storyboard frames** from shot descriptions |
| **Workflow Disconnection** | Worldbuilding and screenwriting are separate processes | Single tool: worldbuild → write → export to production |

**AI Storyboard Generation** is a key differentiator - no competitor (including Clapperbie) offers this.

### Production-Integrated Tools

**Clapperbie** - The standout bridging worldbuilding + production:
- ✅ Industry-standard screenplay formatting
- ✅ Built-in storyboard creation
- ✅ Shot list generation from scenes
- ✅ Scene breakdowns, props lists
- Character/location/props database
- Import from Final Draft, export to industry formats

**StudioBinder** - Strong production, lighter worldbuilding:
- ✅ Screenplay formatting (Final Draft, Fountain, PDF)
- ✅ Storyboard and shot list tools
- ✅ Production docs (call sheets, schedules)
- Script outlining and breakdown
- Less focus on worldbuilding/lore

### Industry Standard Formats

| Format | Usage |
|--------|-------|
| PDF | Universal sharing standard |
| Final Draft (.fdx) | Professional screenwriting |
| Fountain | Plain text screenplay |
| Celtx | Cloud-based screenwriting |

### Vixio Opportunity (Live-Action)

Most worldbuilding tools stop at prose export. The path to production is broken:

```
[World Anvil] → DOCX → [Manual conversion] → [Screenplay software] → [Production tool]
     ↑                        ↑                      ↑
   Friction               Friction               Friction
```

**Vixio approach**:
```
[Vixio] → Screenplay / Storyboard / Shot List (direct from assets)
```

## Market 3: Animation Production

Animation production lacks integrated pre-production tools.

### The Gap

**No Integrated Professional Tool for Animation Pre-Production**
- No Clapperbie/StudioBinder equivalent for animation
- Creators switch between World Anvil, Storyboard Pro, spreadsheets
- Information fragmentation between worldbuilding and production

### Three Workflow Disconnections

| Gap | From → To | Problem |
|-----|-----------|---------|
| **Gap 1** | Written Settings → Visual Settings | Character bios can't auto-generate design briefs for artists |
| **Gap 2** | Static Designs → Dynamic Storyboards | Design images can't be dragged as reusable storyboard assets |
| **Gap 3** | Storyboards → Animation Assets | Can't auto-generate animator production instructions |

### Animation-Specific Production Documents

Animation requires specialized documents that no worldbuilding tool generates:

| Document | Purpose |
|----------|---------|
| Character Design Brief | Visual requirements for character designers |
| Exposure Sheet | Frame-by-frame camera and timing instructions |
| Animation Timing Chart | Spacing and timing for in-betweens |
| Background Art Brief | Requirements for background artists |

### Why Visual Asset-Driven Wins Here

Animation production **thinks in assets**. Vixio's approach matches their mental model:

| Animation Need | Vixio Solution |
|----------------|----------------|
| Reusable storyboard elements | Global asset library |
| Character design briefs | AI-generated from character assets |
| Production documents | Export from assets directly |
| Consistent characters across frames | AI storyboard generation with asset awareness |
