# Feature Design: Delivery Exports

## Overview

Exports are the final stage of the Studio workflow.

They should package the outputs of:

- boards
- canon
- assets

into useful handoff materials.

## Export Principles

### 1. Workflow-Derived

Exports should be generated from the coordinated workspace, not from isolated forms or one-off prompts.

### 2. Review-Friendly

Exports should help users pitch, review, and communicate the work.

### 3. Handoff-Oriented

Exports should make downstream collaboration easier, even before full production integration exists.

## First Exports

### Project Bible

Markdown or document-style export combining:

- project summary
- canon highlights
- asset references
- story context

### Raw Project Data

JSON export for portability and future integrations.

## Next Exports

- scene breakdowns
- shot-oriented exports
- storyboard-ready packages
- screenplay-oriented outputs

## Positioning

Old framing:

> export your world

New framing:

> package your project for review or handoff

That language shift matters because it reinforces the product identity all the way to the end of the workflow.
