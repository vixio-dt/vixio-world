# Vixio Studio - Product Documentation

This folder contains the rewritten strategy, IA, and implementation specs for the Vixio Studio pivot.

## Core Concept

**AI-assisted, visualization-first preproduction workspace**

> The product starts from a project, not a world.

Vixio Studio helps creators move from a concept brief or script into a coordinated preproduction workflow:

- `Planning` - project brief, goals, references, constraints
- `Ideation` - concept directions, style references, alternatives
- `Scripting / Breakdown` - stories, beats, scenes, shot intent
- `Design` - characters, locations, props, visual inputs
- `Storyboard` - shot planning and storyboard-ready outputs

The key shift is that **worldbuilding becomes project canon**, a durable support layer for visualization and downstream delivery instead of the primary product identity.

## Quick Reference

### Product Shift

| Before | After |
|--------|-------|
| Worldbuilder-first | Project-first |
| Entity encyclopedia | Stage-aware workspace |
| Import as hero | Visualization and coordination as hero |
| Worldbuilding as product identity | Canon as support layer |
| Export as utility | Export as final workflow stage |

### Entry Paths

| Path | When To Use |
|------|-------------|
| Project brief | Early concept work, pitches, original ideas |
| Script breakdown | Existing scripts, treatments, scene drafts |
| Reference intake | Lookbooks, notes, decks, supporting canon |

### Flagship Vertical

The product is now **general preproduction first**, with animation-style board workflows used as the strongest demonstration of the system. Animation remains the flagship reference vertical, not the only one.
