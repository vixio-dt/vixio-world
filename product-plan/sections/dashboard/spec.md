# Overview Section Specification

## Overview

The Overview page is the project control room. It should give creators a clear answer to:

- what project am I in?
- where is the work right now?
- what should I do next?

## User Stories

### As a creator, I want to:

1. Understand the current state of my project at a glance
2. Jump quickly into the next meaningful stage
3. See whether boards, canon, assets, and exports are ready
4. Avoid feeling trapped inside entity CRUD as the main product experience

## Main Components

### Project Summary

- project title
- genre / tone
- logline or brief summary
- active CTAs

### Workflow Surface Cards

- Boards
- Canon
- Assets
- Exports

Each card should explain:

- what the surface is for
- how much content exists there now
- why the user should go there next

### Entry Path Reminder

Overview should reinforce that:

- project brief and script breakdown are equal entry points
- canon is supportive, not dominant
- exports are the end of the flow

## Content Strategy

The page should speak in `project`, `boards`, `canon`, and `assets` language.

Avoid describing the workspace as:

- a world
- a lore database
- a general dashboard with unrelated sections

## First Implementation Scope

The first implementation can be a lightweight summary page backed by:

- project metadata
- story counts
- scene / shot readiness
- asset counts
- links into the new IA

The deeper workflow logic can follow later.
