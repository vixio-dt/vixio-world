# Dashboard Section Specification

## Overview

The Dashboard is the entry point to the world, providing an at-a-glance overview of world health, recent activity, and quick access to common actions.

---

## User Stories

### As a worldbuilder, I want to:
1. See how complete my world is at a glance
2. View recent changes and additions
3. Be alerted to consistency issues that need attention
4. Quickly add new elements without navigating away
5. Access my most-used sections easily

---

### Components

#### WorldHeader
- World name (editable inline)
- Genre and tone badges
- Quick add button (dropdown with element types)
- World settings link

#### StatCard
- Icon for element type
- Count of elements
- Click navigates to that section
- Subtle background color per type

#### StatsGrid
- 6 stat cards in responsive grid
- 3 columns on desktop, 2 on tablet, 1 on mobile

#### WarningsCard
- Shows consistency warnings
- Each warning is clickable Ã¢â€ â€™ navigates to affected element
- Collapsible if many warnings
- Badge count in header

#### ActivityFeed
- Chronological list of recent changes
- Icon + action + element name + timestamp
- Click navigates to element
- Load more button for history

#### QuickAdd (Dropdown)
- Options: Character, Location, Organization, Event, Item, Rule, Story
- Opens creation modal or navigates to create page

## Data Requirements

### World Summary
- `name`: World name
- `genre`: Primary genre
- `tone`: Overall tone
- `themes`: Array of themes
- `logline`: Brief description

### Element Counts
- `characters`: number
- `locations`: number
- `organizations`: number
- `events`: number
- `items`: number
- `rules`: number
- `stories`: number
- `scenes`: number
- `shots`: number

### Consistency Warnings
- Array of warning objects:
  - `id`: Warning ID
  - `type`: "conflict" | "gap" | "incomplete"
  - `severity`: "blocker" | "warning" | "note"
  - `message`: Human-readable description
  - `element_type`: Affected element type
  - `element_id`: Affected element ID
  - `element_name`: Affected element name

### Recent Activity
- Array of activity objects:
  - `id`: Activity ID
  - `action`: "created" | "updated" | "deleted"
  - `element_type`: Element type
  - `element_id`: Element ID
  - `element_name`: Element name
  - `timestamp`: ISO date string
  - `user_id`: User who made change (for future multi-user)

## User Flows

### View World Overview
1. User navigates to Dashboard (default route)
2. Dashboard loads world summary, counts, warnings, activity
3. User sees complete overview of world state

### Check Consistency Warnings
1. User sees warnings card with count
2. User clicks a specific warning
3. Navigates to affected element with warning highlighted
4. User resolves issue
5. Warning disappears from dashboard

### Quick Add Element
1. User clicks "+ Add" button
2. Dropdown shows element types
3. User selects type
4. Modal opens for quick creation
5. User fills minimal required fields
6. Saves Ã¢â€ â€™ element created, activity logged
7. Stat count updates

### Navigate to Section
1. User clicks stat card (e.g., "Characters: 24")
2. Navigates to Characters section

## Empty States

### New World (No Elements)
- Welcoming message
- Guide to adding first elements
- Suggested starting points (Characters, Locations, Rules)
- "Getting Started" checklist

### No Warnings
- Success message: "World is consistent"
- Green checkmark icon

### No Recent Activity
- Message: "No recent activity"
