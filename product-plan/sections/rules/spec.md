## User Stories

### As a worldbuilder, I want to:
1. Define clear rules about how my world works
2. Organize rules by category (physics, magic, social, etc.)
3. Check if new content violates existing rules
4. See which rules apply to specific scenarios
5. Track exceptions to rules

## Data Requirements

### Rule
- `id`: UUID
- `world_id`: Parent world
- `code`: Unique identifier (e.g., "MAGIC-003")
- `name`: Rule name
- `category`: physics | magic | technology | biology | social | political | economic | temporal | cosmological
- `statement`: Clear rule text
- `scope`: Where/when applies
- `exceptions`: Array of known exceptions
- `consequences`: What happens when violated
- `examples`: Array of illustrative examples

## Components

### RuleCard
- Rule code badge (color by category)
- Rule name
- Truncated statement
- Category indicator

### RuleDetail
- Full rule display
- Code, name, category
- All fields in readable format
- Edit/delete actions

### RuleForm
- Category selector
- Auto-generated code suggestion
- Rich text for statement
- Array inputs for exceptions, examples

### ConsistencyChecker
- Input field for statement to check
- Check button
- Results display showing applicable rules
- Conflict indicators
