# Characters Section Specification

## Overview

The Characters section manages all people, creatures, and sentient entities in the world. It provides a comprehensive view of character profiles with full relationship mapping and production-ready output for casting and MetaHuman creation.

---

## User Stories

### As a worldbuilder, I want to:
1. View all characters in my world with filtering and search
2. Create detailed character profiles with full backstory
3. Define relationships between characters
4. Track character locations and organization memberships
5. Export character specs for MetaHuman creation

## Data Requirements

### Character
- `id`: UUID
- `name`: Full name
- `aliases`: Array of alternative names
- `role`: protagonist | antagonist | supporting | background
- `species`: Species or type
- `appearance`: Physical description
- `personality`: Personality traits
- `background`: History and origin
- `motivations`: Goals and driving forces
- `arc_potential`: Character development potential
- `visual_references`: Notes for visual design
- `voice_notes`: Speech patterns, accent
- `movement_notes`: Physicality, gestures
- `quote`: Character tagline or signature quote

### Relationships (junction table)
- `character_a_id`: First character
- `character_b_id`: Second character
- `type`: ally | enemy | family | romantic | mentor | rival | acquaintance
- `description`: Relationship details

### Character-Organization (junction)
- `character_id`
- `organization_id`
- `role`: Role within organization
- `status`: current | former | founding

### Character-Location (junction)
- `character_id`
- `location_id`
- `type`: resident | birthplace | workplace | frequents
