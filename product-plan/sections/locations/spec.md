## Data Requirements

### Location
- `id`: UUID
- `world_id`: Parent world
- `parent_location_id`: Parent location (nullable for top-level)
- `name`: Location name
- `type`: world | continent | country | region | city | district | building | room | landmark | natural
- `description`: Physical description
- `atmosphere`: Mood and feeling
- `climate`: Weather and environmental conditions
- `key_features`: Notable landmarks and characteristics (array)
- `history`: Historical significance
- `cultural_significance`: Cultural meaning
- `visual_references`: Notes for environment design
- `lighting_notes`: Lighting conditions
- `sound_notes`: Ambient sound design
- `asset_requirements`: Props and set pieces needed (array)

### Location Hierarchy
- Locations form a tree structure via `parent_location_id`
- Query can fetch full ancestry path
- Sub-locations are direct children

### Character-Location Junction
- `character_id`
- `location_id`
- `type`: resident | birthplace | workplace | frequents

## Components

### LocationTree
- Expandable/collapsible tree structure
- Icons indicate location type
- Click to select, double-click to expand
- Drag-and-drop for reorganization (future)

### LocationCard (for flat list)
- Location name and type
- Parent location breadcrumb
- Atmosphere indicator
- Click to navigate

### LocationDetail
- Full page with tabs
- Sub-locations list
- Character inhabitants
- Production specs

### LocationForm
- Hierarchical parent selector
- Type dropdown
- Rich text for descriptions
- Array inputs for features and assets
